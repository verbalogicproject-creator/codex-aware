# Privacy

Codex Aware processes application-owned structural metadata so authorized actors can resolve context and request declared actions.

Stored data may include logical repository URIs, symbol names, command schemas, graph relationships, content hashes, bounded action arguments, policy decisions, test status, and causal receipt identifiers.

It does not intentionally store raw source bodies, environment files, API keys, provider credentials, raw audio, unrestricted model output, or absolute filesystem paths.

Public demonstration workspaces use sanitized, non-personal fixtures. Pairing tokens are shown once and stored server-side only as hashes.
